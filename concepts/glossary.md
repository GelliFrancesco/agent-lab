# Glossary of Key Terms

Quick reference for vocabulary you'll encounter when working with Claude and agentic AI systems.

---

## A

**Agent**
A system in which an AI model can take actions, observe results, and reason iteratively toward a goal — rather than just generating a single response. See `agentic_ai_overview.md` for a full explanation.

**Agent loop** (also: ReAct loop)
The core cycle of an agentic system: observe → reason → act → observe → ... until the task is complete.

**Agentic AI**
A category of AI systems characterized by autonomy, multi-step reasoning, and the ability to take actions in the world (via tools, code execution, API calls, etc.).

**Anthropic**
The AI safety company that created Claude. Founded in 2021 by former OpenAI researchers.

**API (Application Programming Interface)**
A way for software to talk to other software. The Claude API lets your code send messages to Claude and receive responses, programmatically and at scale.

---

## C

**Claude**
Anthropic's family of AI models. The main model lines are Haiku (fast, efficient), Sonnet (balanced), and Opus (most capable). Each has different speed/cost/capability trade-offs.

**Claude API**
The programmatic interface for interacting with Claude models. Accessed via HTTP or Anthropic's SDKs (Python, TypeScript). Docs: https://docs.anthropic.com/en/api

**Claude Code**
Anthropic's agentic coding tool — a command-line tool that lets developers delegate coding tasks to Claude directly from their terminal. Different from the Claude API; it's a product built on top of it.

**Context window**
The total amount of text (input + output) that a model can process in a single API call. Measured in tokens. Claude supports up to 200K tokens for some models. Everything outside the context window is invisible to the model.

**Conversation history**
The record of all messages in a session — user messages, assistant responses, tool calls, and tool results — that gets included in the context window so the model can refer back to it.

---

## E

**Embedding**
A numerical representation of text as a vector (a list of numbers). Embeddings capture semantic meaning, making it possible to find text that is similar in meaning even if worded differently. Used in retrieval systems (RAG).

---

## F

**Function calling**
Another term for tool use. The model "calls" a function by specifying its name and arguments in a structured format; your code executes the actual function and returns the result.

---

## G

**Grounding**
Connecting an AI model's responses to real, verifiable information — such as retrieved documents, database results, or current web content. Reduces hallucination.

---

## H

**Hallucination**
When a model confidently states something that is factually incorrect or fabricated. Happens because models predict plausible-sounding text, not verified truth. Grounding and tool use can help mitigate this.

**Human-in-the-loop (HITL)**
A system design pattern where a human is asked to review or approve certain agent actions before they are executed — especially for high-stakes or irreversible operations.

---

## I

**Inference**
The process of running a trained model to generate output. When you call the Claude API, you're requesting inference.

---

## L

**Latency**
The time between sending a request and receiving a response. Important for user-facing applications. Faster models (like Haiku) have lower latency; more capable models (like Opus) may be slower.

**LLM (Large Language Model)**
A class of AI model trained on large amounts of text to predict and generate language. Claude, GPT, Gemini, and Llama are all LLMs.

**Long-horizon task**
A task that requires many steps, possibly over an extended period of time. Managing context, state, and error recovery becomes critical for long-horizon tasks.

---

## M

**MCP (Model Context Protocol)**
An open standard created by Anthropic for connecting AI models to external tools and data sources in a standardized way. Allows tools, resources, and prompts to be plugged into Claude via a common interface. Docs: https://modelcontextprotocol.io

**Memory** (in agents)
How an agent retains information. Types include: in-context (in the current window), external/retrieval (a database), in-weights (baked into the model), and cached (pre-computed).

**Multi-agent system**
An architecture where multiple AI agents collaborate, with one agent orchestrating others or agents working in parallel on different subtasks.

---

## O

**Orchestrator**
In a multi-agent system, the orchestrator is the "top-level" agent that receives the overall task, breaks it down, and delegates to subagents or tools.

**Output token**
A single token generated by the model as part of its response. API pricing is typically based on input tokens + output tokens.

---

## P

**Prompt**
The input you send to a language model. Can include instructions, examples, context, conversation history, and more.

**Prompt engineering**
The practice of designing effective prompts to get the best possible results from a language model. A real skill with meaningful impact on output quality.

**Prompt injection**
An attack where malicious content embedded in data the agent reads (a web page, a file, an email) tricks the agent into taking unintended actions. A significant safety concern for agents that process external content.

---

## R

**RAG (Retrieval-Augmented Generation)**
A technique where relevant information is retrieved from an external knowledge base and included in the prompt context at query time. Allows the model to answer questions about content it wasn't trained on, and reduces hallucination.

**ReAct**
A prompting pattern for agents: Reason, then Act. The model explicitly thinks through what it should do (reasoning) before taking an action. Tends to improve reliability compared to acting without explicit reasoning.

---

## S

**Streaming**
Receiving model output token-by-token as it's generated, rather than waiting for the full response. Makes UIs feel more responsive.

**Subagent**
In a multi-agent system, a specialized agent that handles a specific type of task, called by the orchestrator.

**System prompt**
A special message sent at the start of a conversation that sets instructions, persona, and context for the model. Separate from the user message and typically not visible to end users in production apps.

---

## T

**Temperature**
A parameter controlling how "random" or "creative" the model's output is. Low temperature (close to 0) = more deterministic; high temperature (close to 1) = more varied. For agentic tasks that require precision, lower temperature is often better.

**Token**
The basic unit of text that models process. Roughly ¾ of a word on average (for English). "Hello world" is about 2 tokens. Models have limits on how many tokens they can process per request.

**Tool**
A capability you give to an agent that lets it take an action — like searching the web, running code, reading a file, or calling an API. Tools are defined by you and executed by your code; the model just decides when and how to call them.

**Tool use** (also: function calling)
The mechanism by which a model requests that a specific tool be called with specific arguments. The model produces a structured "tool call," your code runs it, and the result is fed back to the model.

---

## V

**Vector database**
A database optimized for storing and searching embeddings (vectors). Used in RAG systems to efficiently find the most relevant documents for a given query. Examples: Pinecone, Weaviate, pgvector.
